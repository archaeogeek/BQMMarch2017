Template Servers

making iSIC deployment simple

Jo | Astun BQM March 2017

Why?

To make it as easy as possible to deploy new iSIC implementations, saving time for developers and the delivery team

iSIC = iShare in da the Cloud \m/

How?

User stories to talk through pain-points for Devs, Delivery, PM, Boss, Client

(in no particular order)

Aim

Pre-configure as much as possible in the template servers, so that deployment is quick and painless

What?

iShareMaps and GIS now on separate servers, iShareMaps now on a single server (*), SDW templates for PostGIS 1.5 and 2.2

More on this later

Major Design Changes

No version numbers or client names anywhere

All customer-specific configuration is on the data drive and symlinked from the standard locations

Data

Ready-configured with ADS Base Mapping

Boundary-line, EduBase, Police, NHS Choices ready-downloaded

Uses AddressBase for AstunLocationLookup

Maps

Web and WebService back on the same server (webservice on a separate website running on a non-default port)

Updated responsive design template based on astuntechnology.com(*)

Beta

GIS

Restricted mapsource set up for testing editing and security

Both

Scheduled tasks set up for database backup and mapserver maintenance

Uses Amazon SES for workflow notifications and other email

Scripts for getting map extents and centroids for configuring mapsources

QGIS and other utility software installed

Techy stuff

Amazon CloudWatch alarms set up for monitoring disk drives, memory, performance

Disk quotas for all drives, with email notification

Delivery Tasks

"Personalise"- download correct AddressBase, set ADS endpoints, set extents for maps

"Secure"- change defaults passwords and add client users

Currently some minor yak-shaving to connect to SDW and set identifiable machine name
